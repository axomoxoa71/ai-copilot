Overview

The Agent API is a RESTful service that orchestrates multi-agent conversations using LangGraph. It intelligently routes requests between specialized agents based on content analysis and delegation rules.

Base Technology Stack:

  • Express.js REST framework
  • LangGraph for agent orchestration
  • OpenAI/OpenRouter LLM provider
  • OpenTelemetry for observability
  • Zod for request validation

Base URL

http://localhost:8787

For production deployments, update the base URL accordingly. The API port is configurable via the AGENT_API_PORT environment variable.

Authentication

Current Status: No authentication required (suitable for local development).

Production Recommendation: Implement bearer token or API key validation before deploying.

Chat Completion Endpoint

POST /agent-api

Purpose: Submit a user message and receive agent response with intelligent routing.

Request Headers

POST /agent-api HTTP/1.1
Host: localhost:8787
Content-Type: application/json

Request Body Parameters

sessionId required

Unique session identifier (non-empty string). Used to group related messages together.

userPrompt required

User message to send to the agent (non-empty string).

Request Body Example

{
  "sessionId": "unique-session-identifier",
  "userPrompt": "What are the open Jira tickets in the current sprint?"
}

Response (Success)

200 OK

{
  "agentResponse": "Based on the query, I've found...",
  "agentName": "atlassian-agent",
  "sessionId": "unique-session-identifier",
  "tokenUsage": {
    "prompt_tokens": 250,
    "completion_tokens": 150,
    "total_tokens": 400
  },
  "logs": [
    {
      "timestamp": "2026-06-19T10:30:45.123Z",
      "component": "delegationEngine",
      "message": "Routed to atlassian-agent",
      "level": "info"
    }
  ],
  "traceparent": "00-0af7651916cd43dd8448eb211c80319c-..."
}

Response (Validation Error)

400 Bad Request

{
  "error": "Invalid request payload.",
  "details": {
    "userPrompt": ["String must contain at least 1 character"]
  }
}

Request/Response Schemas

Chat Completion Request

Field Type Required Description
sessionId string Yes Unique session identifier (min 1 character)
userPrompt string Yes User message (min 1 character, max 1MB payload)

Chat Completion Response

Field Type Description
agentResponse string The agent's response text
agentName string Name of the agent that handled the request
sessionId string Echo of the request session ID
tokenUsage object Token consumption metrics (prompt, completion, total)
logs array Structured logs from request processing
traceparent string W3C Trace Context for correlation

Code Examples

cURL Example

curl -X POST http://localhost:8787/agent-api \
  -H "Content-Type: application/json" \
  -d '{
    "sessionId": "user-123-session-456",
    "userPrompt": "What is 2 + 2?"
  }'

JavaScript/TypeScript

async function sendMessage(sessionId, userPrompt) {
  const response = await fetch(
    'http://localhost:8787/agent-api',
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ sessionId, userPrompt }),
    }
  );
  
  if (!response.ok) {
    throw new Error(await response.text());
  }
  
  return response.json();
}

const result = await sendMessage(
  'my-session',
  'Show me JIRA tickets'
);
console.log(result.agentResponse);

Python

import requests

response = requests.post(
    'http://localhost:8787/agent-api',
    json={
        'sessionId': 'my-session',
        'userPrompt': 'What are open issues?'
    }
)

data = response.json()
print(data['agentResponse'])
print(f"Tokens: {data['tokenUsage']['total_tokens']}")

Error Handling

Status Meaning Common Causes
200 OK Request succeeded
400 Bad Request Invalid payload, missing required fields
500 Internal Error Unhandled server exception, LLM provider error

All errors include structured JSON response with error message and optional details field for validation errors.

Monitoring & Observability

Health Check Interval

Recommended: Check health endpoint every 30 seconds

Metrics to Monitor

  • Response Time: p50, p95, p99 latency
  • Error Rate: 4xx and 5xx response percentage
  • Token Usage: Average tokens per request
  • Active Sessions: Concurrent session count
  • LLM Latency: Time to receive LLM response

Trace Correlation

All responses include traceparent header following W3C Trace Context. Use this to correlate logs across services:

  • Frontend request logs
  • Backend processing logs
  • LLM API logs
  • External service calls
